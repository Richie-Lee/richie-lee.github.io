PCA Machine Learning Hybrid Models: Real Estate Forecasting

Mining big data using parsimonious factor, machine learning, variable selection and shrinkage methods

Posted by Richie on Sunday, June 6, 2021

TL:DR

The objective of this study is to predict Dutch House prices using macroeconomic variables that were collected from Government open-source databases. Given the high number of explanatory variables relative to the number of observations, we recognise a good fit with parsimonious modelling. These models are characterised by their great explanatory power from minimal resources and are especially valuable in cases of limited or low-frequency data.

The methods considered follow Kim & Swanson (2018) hybrid model design, where we combine factor modelling techniques (PCA, ICA, Sparse PCA) and machine learning ensembles (Boosting) and regularization methods (Elastic Net, Ridge regression, LASSO). Overall, the best results were indeed produced by Machine Learning combined with factor analysis, where model selection appeared to be strongly case-dependent. Mean model combinations as a solution provided the overall best results, improving performance up to 25% from regular implementations.

Context

In recent years or even decades, there has been a growing recognition of the importance of the interactive connection between the real-estate market and the macroeconomy Leung (2004). However, as something that is driven by numerious aspects, it is not easy to model. For this purpose, we employ PCA Machine Learning hybrid models alongside various variations to tackle common challenges in this context such as high dimensionality, low-frequency data, multicollinearity, biases and complex non-linear relations.

Data

The data was sourced from a government database (CBS) and includes key macroeconomic indicators covering various sectors of the economy. After eliminating irrelevant variables, 57 variables (N) were retained, reported over 88 quarters (T) spanning from the first quarter of 1999 to the last quarter of 2020. The target variables were various indices of Dutch Housing prices.

To enhance data quality, the following considerations were taken into account:

  1. Non-stationary: Through economic intuition and exploratory analysis, a significant trend and Seasonality were identified, leading to variations in mean and variance over time. To address this, the target variable was transformed as follows: y_growth_rate, t = log(y_t / y_{t-4}) * 100%.The values that it produces are interpreted as the growth rate relative to the previous year expressed in percentages.

  2. Standardization: Variables measured on different scales could introduce biases during analysis. To mitigate this issue, all variables were rescaled to have a mean of 0 and a standard deviation of 1 as follows: x_std = (x - mu_x) / sigma_x. This preserves relative differences while removing magnitude biases.

  3. Missing Values: Care was taken not to impute missing values with future data or aggregates containing future information, as it can lead to data leakage and increased risk of overfitting.

  4. Multicollinearity: Variables with perfect correlation were identified and removed to avoid identification problems in regression designs.

Train-test split

With the exception of the autoregressive benchmark model, all models require tuning. For this purpose, a train/validation/test split is implemented alongside an expanding window (also known as recursive estimation). This method uses a consistent validation size (5 quarters), a consistent testing size (1 quarter), and increases the training size by one observation with every iteration. Methodology

Given the high number of variables (N = 57) and the relatively low number of observations (T = 88) we recognise a good fit with parsimonious modelling. These models are characterised by their great explanatory power from minimal resources and are especially valuable in cases of limited or low-frequency data.

The methods applied here closely follow Kim & Swanson (2018)’s design, in which factor analysis is combined with machine learning. More specifically, hybrid models are constructed by first applying principle component analyses and then using these factors as input data for 1-step ahead machine learning forecasts. All hyperparameters were tuned using Grid Search.

Machine Learning

Boosting

Gradient boosting is an ensemble machine learning technique used for classification and regression modeling. It builds ensembles of decision tree models to correct prediction errors iteratively. One prominent implementation of gradient boosting developed by Chen and Guestrin (2016) is Extreme Gradient Boosting (XGBoost), which offers a balance between computational efficiency and performance. This implemenation considers the following hyperparameters:

  1. n_estimators: Influences the number of trees in the ensemble and the iterations during model fitting.

  2. max_depth: Controls the complexity of individual trees by setting the maximum depth of decision trees.

  3. eta (learning rate): Weighs the contribution of each model, impacting the overall influence of each tree on the final prediction.

  4. subsample: Introduces randomness by specifying the number of samples (rows) used in constructing each tree.

  5. n_features: Defines the number of features (columns) considered in building each tree, potentially enhancing model robustness.

It’s important to be mindful that randomness plays a role in constructing the XGBoost model. Consequently, running the algorithm multiple times on the same data might lead to slight variations due to this inherent stochastic nature.

Elastic Net

Elastic Net (EN) is an extension of linear regression in machine learning that incorporates regularization penalties into the loss function, encouraging simpler models with smaller coefficient values. Specifically, it combines both L2 (Ridge regression) and L1 (LASSO) penalty functions.

Ridge regression uses L2 penalties, which introduce shrinkage estimators (ridge estimators) to push the estimations closer to the “true” population parameters. This is particularly useful in the presence of multicollinearity (highly correlated explanatory variables). Additionally, ridge regression does not require the number of observations (T) to exceed the number of parameters (N). Instead of omitting variables, ridge regression allows the coefficients to converge to 0 as their relevance decreases.

Elastic Net also incorporates L1 penalties, similar to LASSO (Least Absolute Shrinkage and Selection Operator). However, unlike ridge regression, LASSO omits variables entirely instead of just converging their coefficients to 0. It is especially valuable in handling estimations with many redundant (highly correlated) variables, tending to retain only one variable from correlated parameter sets.

The Elastic Net method combines both penalty functions and introduces tunable hyperparameters. The specific Elastic Net specification considered in this paper is as follows:

P_{EN} = (\alpha * P_{L1}) + ((1 – \alpha) * P_{L2})

\mathcal{L}_{EN} = \mathcal{L}_{OLS} + \lambda * P_{EN}

Here, $P_{{L1, L2, EN}}$ denotes L1, L2, and Elastic Net penalty functions, respectively. $\mathcal{L}_{{EN, OLS}}$ refers to the loss functions for Elastic Net and Ordinary Least Squares (OLS). The hyperparameters considered here include:

  1. alpha: weights the relative contributions of L1 and L2 regularization (e.g., $\alpha =$ 1 implies using LASSO only)
  2. lambda: controls the combined weight of the penalty functions.

Factor analysis

PCA: Principal Component Analysis

Principal Component Analysis (PCA) is a well-established method that seeks to reduce dimensions while preserving as much information as possible. After taking the original variables as inputs, PCA derives a set of independent latent variables which are ordered by captured variance. These latent variables are also known as principal components (denote $k$ = number of components used). The underlying idea is that the first $k$ principal components would capture the majority of the information, under the assumption that the components that capture the most variance would be the most informative (generally, but not necessarily true). It is then a matter of optimizing the trade-off between model simplicity (by reducing $k$) and total information (by increasing $k$)

Reducing dimensionality improves both computational efficiency and overfitting, as these aspects suffer from large sets of variables. This is especially true for machine learning algorithms. By eliminating correlations amongst the components, PCA also improves performance and visualization options. However, in return for these benefits, PCA sacrifices information contained within the variables as well as their interpretability.

ICA: Independent Component Analysis

This method can be considered an extension of PCA, which is commonly used in signal processing use-cases (example). Its primary goal is to extract hidden factors, often referred to as sources, from the data by transforming it into maximally independent components. To achieve this, two conditions need to be satisfied: (1) the data should be statistically independent and (2) non-Gaussian, as this is necessary for identifying independence.

In terms of specific data preprocessing for Independent Component Analysis (ICA), we recommend pre-whitening the data through centering and whitening. Additionally, it is advisable to omit potentially normally-distributed input variables, for which we used the Jarque-Bera Normality test. For the implementation of ICA, the “FastICA” package is used, known for its efficiency and popularity Hyvarinen (1999).

SPCA: Sparse Principal Component Analysis

Sparse Principle Component analysis (SPCA) can in broad terms be considered a PCA optimization problem (reconstruction error minimization) that cleverly uses L1 penalties on the components to introduce sparsity, which contribute to reducing noise when handling few training samples. SPCA yields a more parsimonious and interpretable representation. that clearly emphasizes which original features contribute to the difference between samples.

Conclusion

Both Boosting and Elastic Net showed significant performance improvements when compared to traditional principal component regression (PCR), with a preference for Boosting. However, despite its seemingly overall inferior explanatory power, PCR (OLS) still frequently produces the most-accurate predictions. For this reason, machine learning methods are suggested to not entirely replace traditional factor models, but to be regarded as promising case-dependent alternatives. A related finding supporting this “case-dependency”, is the fact that models displayed “clustered” periods in which they outperformed the alternatives, hinting at periods in which certain conditions favored certain models (Figure 3). In general, machine learning models were found to be more useful for data sets with weaker trends and volatility
